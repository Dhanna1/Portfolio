PROJECT OVERVIEW

A connected keychain designed around shared anticipation.

Calendate is a pair of compact devices intended to show the next shared date, days remaining, messages, reminders and an animated digital companion.

DEVELOPMENT TIMELINE

Product refinement through repeated enclosure and interface decisions.

01

Define the relationship concept

Established the date countdown, short messages and paired-device idea.

02

Select compact hardware

Chose the XIAO ESP32-C3, OLED, one-button input, LiPo power and charging hardware.

03

Develop the interface

Implemented screens, reminders, button navigation and the animated companion.

04

Iterate enclosure geometry

Completed more than eight revisions to improve fit, button position, USB access and overall size.

05

Prepare physical integration

The remaining work is soldering inside the confined package, loading the ESP32 and connecting cloud synchronization.

INTERFACE MILESTONE

Standalone behavior demonstrated before final hardware assembly.

PROBLEMS & ITERATIONS

The greatest difficulty is not the screen—it is packaging a complete product.

01

Confined wiring

The final device requires reliable soldered connections within a very small enclosure.

02

Enclosure tolerance

Small changes to the OLED, button, battery, charging board and USB slot affected overall fit.

03

Cloud dependency

The standalone experience works, but paired updates require the ESP32 and cloud workflow to be completed together.

VERSION 1 DEFINITION

A finished Version 1 will be a fully enclosed, battery-powered pair of devices.

Completion requires final soldered hardware, code running on the ESP32, button-controlled screens, battery validation and synchronized date/message updates between the two devices.
